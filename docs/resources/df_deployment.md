# cdp_df_deployment (Resource)

Initiates a CDP DataFlow deployment on an enabled DataFlow service. A deployment runs a NiFi flow definition on the DataFlow service's Kubernetes cluster.

## Example Usage

```terraform
//
// This file is distributed on an "AS IS" BASIS, WITHOUT WARRANTIES OR CONDITIONS
// OF ANY KIND, either express or implied. Refer to the License for the specific
// permissions and limitations governing your use of the file.

terraform {
  required_providers {
    cdp = {
      source = "cloudera/cdp"
    }
  }
}

# Basic deployment
resource "cdp_df_deployment" "basic" {
  service_crn      = cdp_df_service.example.crn
  flow_version_crn = cdp_df_flow_definition.example.flow_version_crn
  deployment_name  = "my-deployment"
}

# Full deployment with all options
resource "cdp_df_deployment" "full" {
  service_crn      = cdp_df_service.example.crn
  flow_version_crn = cdp_df_flow_definition.example.flow_version_crn
  deployment_name  = "my-full-deployment"

  cluster_size     = "EXTRA_SMALL"
  cfm_nifi_version = "1.21.0"
  auto_start_flow  = false
  project_crn      = cdp_df_project.example.crn

  static_node_count = 1

  # Change flow version strategy (used when flow_version_crn is updated)
  strategy                       = "STOP_AND_PROCESS_DATA"
  wait_for_flow_to_stop_in_minutes = 15

  # Parameter groups as JSON
  parameter_groups = file("parameters.json")

  polling_timeout = 3600
}

# Deployment with auto-scaling
resource "cdp_df_deployment" "autoscaled" {
  service_crn      = cdp_df_service.example.crn
  flow_version_crn = cdp_df_flow_definition.example.flow_version_crn
  deployment_name  = "my-autoscaled-deployment"

  cluster_size         = "SMALL"
  auto_start_flow      = false
  auto_scaling_enabled = true
  auto_scale_min_nodes = 1
  auto_scale_max_nodes = 3
}
```

## Schema

### Required

- `deployment_name` (String) The name of the deployment.
- `flow_version_crn` (String) The CRN of the flow definition version to deploy. Changing this triggers an in-place flow version change.
- `service_crn` (String) The CRN of the DataFlow service where the deployment will be created.

### Optional

- `auto_scale_max_nodes` (Number) Maximum number of nodes for auto-scaling.
- `auto_scale_min_nodes` (Number) Minimum number of nodes for auto-scaling.
- `auto_scaling_enabled` (Boolean) Whether auto-scaling is enabled. Defaults to false.
- `auto_start_flow` (Boolean) Whether to automatically start the flow after deployment. Defaults to true.
- `cfm_nifi_version` (String) The CFM NiFi version to use. Defaults to the latest version.
- `cluster_size` (String) The cluster size. Valid values: `EXTRA_SMALL`, `SMALL`, `MEDIUM`, `LARGE`. Defaults to `EXTRA_SMALL`.
- `parameter_groups` (String, Sensitive) The parameter groups JSON for the deployment. Pass as a JSON string (e.g. `file("parameters.json")`). The content is hidden from plan output; use `parameter_groups_sha` to detect changes.
- `polling_timeout` (Number) Timeout in seconds for polling the deployment status. Defaults to `3600` (1 hour).
- `project_crn` (String) The CRN of the project to assign this deployment to.
- `static_node_count` (Number) The static number of nodes. Defaults to 1. Ignored when auto-scaling is enabled.
- `strategy` (String) The strategy to use during change flow version. Valid values: `STOP_AND_PROCESS_DATA`, `STOP_AND_EMPTY_QUEUES`, `ONLY_RESTART_AFFECTED_COMPONENTS`. Defaults to `STOP_AND_PROCESS_DATA`.
- `wait_for_flow_to_stop_in_minutes` (Number) The max time in minutes to wait for the flow to bleed out during change flow version. Only relevant with `STOP_AND_PROCESS_DATA` strategy. Defaults to 15.

### Read-Only

- `current_node_count` (Number) The current node count.
- `deployed_by_name` (String) The name of the person who deployed the flow.
- `deployment_crn` (String) The CRN of the deployment.
- `flow_crn` (String) The CRN of the flow definition.
- `flow_name` (String) The name of the flow.
- `id` (String) The ID of the deployment, same as the deployment CRN.
- `name` (String) The name of the deployment (from API).
- `nifi_url` (String) The URL to open the deployed flow in NiFi.
- `parameter_groups_sha` (String) SHA256 hash of the parameter_groups content. Changes when the parameter groups file is modified.
- `status` (String) The status of the deployment.
- `status_message` (String) The status message of the deployment.
